Workflow

First detail each element, then focus on grid design. During the grid design, we first focus on the whole website layout, then small field layout.

Rules

If a style will only be used once in the whole website, we will label it as id. If the certain style will be applied for more than once, we will label it as class.

gXxx only used for grid strcutre section.

  • Class

    • Don't use abbreviations
    • Capital the first letter of the word
    • No space
    • You can use - to distinguish words
    • The grid class begins with “g”

  • ID

    • Don't use abbreviations
    • All lowercase
    • No space
    • You can use - to distinguish words
    • img alt: Same as image name

Nesting Convention

We will use section to divide the page content. Once we need a grid to form layout, we use div wrap the field.

We follow Brad Frost's Atomic Design Methodology to determine the hierarchy of our div's classes. We examine the different sections' content and then split them into different atoms, molecules, and organisms to arrange their classes.

Commenting Convention

Using comment to divide codes into sections in css files based on the usage of css code.

Such as:<!-- Xxx section START --> and <!--Xxx section END -->

Adding detailed information for codes that cannot identify the usage based on its name.
